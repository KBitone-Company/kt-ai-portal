# 사용자 관리 설계

## 요구사항 대응 범위

- 본인 인증을 통한 사용자 등록 신청
- 사용자 등록 승인
- 외부 사용자 계정 조회/등록/수정/권한 부여
- 사용자 소속/역할 분류
- 접속 가능 서비스 설정
- 사용자 상태 변경 이력

## 현재 구현 범위

- mock data 기반 화면
- mock workflow (승인/반려/보완요청/재신청 접수)
- 실제 Keycloak write 연동 없음
- 실제 DB 저장 없음

## 화면 구성

### 1. 사용자 등록 신청

- 신청 목록: 신청 ID, 신청자명, 사용자 유형, 소속, 이메일, 연락처, 신청 사유, 본인확인 상태, 승인 상태, 신청일, 희망 접속 기간, 요청 역할, 요청 서비스
- 신청자 선택 시 우측 상세 화면 표시

### 2. 사용자 등록 상세

- 기본정보: 이름, 이메일, 연락처, 소속기관, 부서, 직급/직책, 사용자 유형
- 본인확인 정보: 본인확인 방식, 일시, 결과, 확인자
- 신청 정보: 신청 사유, 업무 목적, 요청 역할, 요청 서비스, 접속 희망 시작일/종료일
- 보안 확인: 보안서약 제출 여부, 개인정보 동의 여부, 외부 사용자 보안 확인 여부
- 승인 처리: 검토 의견 입력, 승인/반려/보완요청/재신청 접수 버튼

### 3. 사용자 관리

- 사용자 목록: 사용자명, 계정 ID, 이메일, 소속, 역할, 유형, 계정 상태, 마지막 로그인
- 사용자 선택 시 소속/역할/접속 가능 서비스 설정 패널 표시
- 사용자별 상태 변경 이력 표시

### 4. 외부 사용자 관리

- 외부 사용자 목록: 사용자명, 이메일, 소속기관, 계약/수행사명, 계정 상태, 접속 가능 시작일/종료일, 부여 역할, 접근 가능 서비스, 마지막 로그인, 만료 예정 여부
- 신규 등록/수정/접속 기간 연장/계정 잠금/비활성화 버튼 (mock)

### 5. 사용자 상태 변경 이력

- 이력 유형: 사용자 등록 신청, 본인확인 완료, 검토 시작, 승인, 반려, 보완요청, 계정 생성, 계정 잠금, 계정 해제, 역할 변경, 접속 기간 변경, 접근 서비스 변경
- 표시 항목: 발생 시각, 처리자, 이벤트, 이전 상태, 변경 상태, 사유, IP/처리 위치

## 후속 Keycloak 연동 방향

- Keycloak Admin API 연동
- 사용자 생성/수정/비활성화
- Role/Group mapping
- Required Action 설정
- Password reset email
- 계정 잠금/해제
- 외부 사용자 유효기간 관리
- 감사로그 연동

## 후속 DB 설계 방향

- 사용자 신청 테이블
- 승인 이력 테이블
- 외부 사용자 계약/기간 테이블
- 접근 가능 서비스 매핑 테이블
- 사용자 상태 변경 이력 테이블
